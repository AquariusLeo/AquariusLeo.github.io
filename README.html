此仓库用于同步个人博客的Hexo的source文件夹。

用法

参考视频：https://www.bilibili.com/video/BV1nY6xYmEw7

  1. 首先安装nodejs：https://nodejs.org/zh-cn
  2. 修改npm源为国内源：npm config set registry https://registry.npmmirror.com
  3. 安装hexo客户端：npm install -g hexo-cli
  4. 初始化博客文件夹：hexo init my-blog
  5. my-blog将作为hexo博客的主目录，跳转目录：cd my-blog
  6. 安装依赖：npm install
  7. 此时my-blog文件夹下应当有一个source文件夹，其作用是存放所有的博客原始文件。清空其内容，把本仓库的内容拉取到此目录下：git clone git@github.com:AquariusLeo/myHexoSource.git ./source
  8. source文件夹有几个_config文件，这些是用来配置hexo以及fluid主题的配置文件，已编辑好。将其拷贝到my-blog目录下

hexo使用方法

所有的博客原始Markdown都在source/_posts下，可使用Typora编辑。编辑完成后在hexo博客的主目录my-blog下执行：

  1. 生成静态文件，每次修改网站时执行一次：hexo generate
  2. 开启本地服务器预览博客，然后可在本地浏览器预览效果：hexo server
  3. 安装git部署的npm包（执行一次）：npm install hexo-deployer-git --save
  4. 发布新博客，同步到git仓库：hexo clean; hexo generate; hexo deploy
  5. 至此，编译好的网站静态文件就会同步到自己的github仓库：https://github.com/AquariusLeo/AquariusLeo.github.io
  6. AquariusLeo.github.io已设置好Github Pages自动部署
  7. 博客网址：https://leo.zz.ac 或 https://aquariusleo.github.io

CNAME

source文件夹下的CNAME文件用于配置上面的博客网址。修改后，需执行hexo clean && hexo generate && hexo deploy，同步到AquariusLeo.github.io仓库才正式生效。
